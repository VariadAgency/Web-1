# Step 4.1 & 4.2: JS Scroll Performance

**Ziel:** Reduzierung der CPU-Last beim Scrollen durch Throttling/Debouncing.
**Betroffene Dateien:** `assets/js/main.js`

## Änderungen
- Einführung eines zentralen `requestAnimationFrame` Patterns für alle Scroll-Events.
- Zusammenfassung mehrerer unabhängiger `scroll` Listener, wo möglich.
- Alternativ: Sicherstellen, dass JEDER Listener das `ticking` Pattern nutzt (was oft schon der Fall war, aber nicht überall).

## Begründung
Vermeidung von "Jank" (Ruckeln) auf Mobilgeräten. Code wird ausgeführt, bevor der Browser den nächsten Frame zeichnet, statt hunderte Male dazwischen.

## Risiken
- Funktionalität könnte brechen, wenn Variablen-Scopes nicht beachtet werden.
